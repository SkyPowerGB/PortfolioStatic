Java swing - CAD/CAM

Project Motivation

During my internship at SpikeFish, there were periods without assigned tasks for students. Since I had no university backlog at the time, I decided to start working on a personal project.

I was still waiting for CNC hardware components, so firmware development was not possible. Instead, I decided to experiment with building a simple 2.5D CAD/CAM application in Java.

Core concept

The main idea was to build a raster-based drawing system combined with depth layers. Each layer represented a specific Z-depth value, enabling a simplified 2.5D workflow.

Each pixel corresponded to a value in millimeters, ideally matching CNC machine resolution. The system included tools such as line, circle, rectangle, Bézier curve, and basic image engraving support.

The overall architecture was structured around the MVC pattern in order to separate rendering, logic, and user interaction.

Technical Implementation

Implemented functionality included zooming and panning, workspace centering, and 1:1 scale reset. Layers were selectable and represented depth levels.

I implemented a basic line drawing tool, a line connection helper that highlights endpoints for easier line joining, and line-point editing which allowed vertex manipulation. A text tool was also added.

To automatically sort layers by depth, I implemented a binary search tree structure.

Architectural Challenges

Although the project initially followed the MVC pattern, over time the main controller accumulated too many responsibilities, which led to tighter coupling and reduced maintainability.

Another major challenge was raster-to-G-code conversion. In hindsight, a vector-based representation would likely have been more suitable for toolpath generation.

The overall scope of the project turned out to be overly ambitious for a single developer at that stage.

Refactoring Attempt

I later attempted to refactor the system by reducing the scope to line-based geometry and exploring vector-style storage to simplify future G-code generation.

However, the refactor introduced regressions in layer handling and line-point editing. Due to shifting priorities, the project remains unfinished in its current state.
