Portfolio

Project goals

Goal of this project was to make my own site where ill post stuff i do. Site was aimed to be static for security reasons .

Js would be used to dynamically load content from content folders. It would also make optional link to the content folder if there is more detailed page about project

While content is loaded it would pass trough filters for an easier shearch.

Implementation details

Site consists of main sites that act as an frame for data.

JS loads content using adressMaps "registers". Each content site has its own set of registers. Register holds list of folders and corresponding filters.

JS fetches register and goes trough list .If item passes filters its folder is used along aditional data from configJson to fetch its metadata.

Metadata contains: Title, summary (which can optionally be in txt file), date, hasSite flag, optional thumbnail. This data is used for building article card and if hassite is true it will add link to folder. Since folder is expected to have index.html card is connected to page.

Reason for registers is to reduce number of fetching if filters dont match.

Other stuff handeled by JS are filters and navbar which are also generated. ConfigJson is used to define them.
