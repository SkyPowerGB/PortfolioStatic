Project overwiev

Project was the university class exam. We were divided into groups and each was given different site type with same goals.

Goal was to make Vheicle repair shop website. Site must have mutlirole users. It must hash passwords ,use secure cookie as sign in method and print reciept in PDF. It must have validation and CRF tokens across requests. It should use pagination for large datasets. It should use AJAX for loading content without refreshing.

Since group was inacitve , authot took initiative and made whole site alone. All functionality goals were met, only failure was design.

Site allows user to register and "appoint" you vheicle to the shop. The mechnanic will update the vheicle status and add services . The admin can manage users add roles and manage services.
